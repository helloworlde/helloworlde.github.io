使用 Fleet Telemetry 让 Tesla 车辆主动上报数据进行监控-(五)数据处理

Tesla 车辆监控：Fleet Telemetry MQTT 数据接收、EMQX 规则引擎解析及 PostgreSQL 数据库存储实战教程。

2025-09-28 | Updated 2026-07-11

  • Tesla

在正确配置 Fleet Telemetry 并下发配置给车辆后，就可以接收到车辆的上报数据，并且通过 MQTT 转发。可以通过服务日志或配置的 datastore 查看车辆上报 records，下面以 logger、MQTT datastore 以及 EMQX 规则引擎写入 PostgreSQL 为例。

一、查看 records

1.1 查看日志

如果 records.V 配置了 logger，可以直接查看 Fleet Telemetry 服务日志。下面日志是电量相关字段的 record：

docker logs -f fleet-telemetry

fleet-telemetry  | {"activity":true,"context":"fleet-telemetry","data":{"CreatedAt":"2025-09-28T00:43:13Z","EstBatteryRange":{"doubleValue":232.09772017025708},"IdealBatteryRange":{"doubleValue":232.09772017025708},"IsResend":false,"RatedRange":{"doubleValue":232.09772017025708},"Soc":{"doubleValue":86.09375},"Vin":"{VIN}"},"level":"info","metadata":{"device_client_version":"1.0.3","receivedat":"1759020194000","timestamp":"0","txid":"ac19f006a71b472296b8b3-000000495","txtype":"V","version":"0","vin":"{VIN}"},"msg":"record_payload","time":"2025-09-28T00:43:14Z","vin":"{VIN}"}

1.2 查看 MQTT topic

如果 records.connectivity 配置了 mqtt，连接状态 topic 为 telemetry/{VIN}/connectivity；车辆字段数据使用 telemetry/{VIN}/v/{field_name} 这种 topic 格式。

{
  "ConnectionId": "xxxx",
  "CreatedAt": "2025-09-27T12:22:21Z",
  "Status": "CONNECTED"
}

MQTT datastore 的 topic 结构由 Fleet Telemetry 实现决定：字段数据、连接状态、警报和错误分别发布到不同 topic，payload 为 JSON。具体格式以官方 MQTT Datastore 为准。

1.3 处理数据

可以使用 EMQX 的规则引擎将数据存储到数据库中，或者使用其他的工具进行处理。下面以将数据存储到 PostgreSQL 为例。

  • 在 PostgreSQL 创建表
create table if not exists tesla_fleet_telemetry (
    id bigint generated always as identity primary key,
    topic varchar(1000),
    payload varchar(10000),
    "timestamp" timestamp,
    add_time timestamp default now()
);
  • 使用 EMQX 规则引擎插入数据
insert into tesla_fleet_telemetry (topic, payload, timestamp)
 values
 (${topic}, ${payload}, TO_TIMESTAMP((${timestamp} :: bigint)/1000));

  • 查看 PostgreSQL 数据
select *
from tesla_fleet_telemetry
order by add_time desc

这里只是简单地将数据存储到数据库中，实际使用时需要对数据的 topic、payload 进行解析，然后存储到对应的字段中，方便后续查询和分析。
